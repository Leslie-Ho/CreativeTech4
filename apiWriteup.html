Process

When looking at the available API’s, the most important aspect of each API had to be one that could either output a value that can be used as an input or one that could use an input to generate an output. I began with the weather API since there were many changing variables that could be pulled such as the temperature, humidity, wind degree, weather description, etc. These were my baseline that would branch out to other possible APIs.
Initially, I wanted to use the weather to create a specialized playlist that couldn’t be replicated without identical weather conditions. The Spotify API allows micro adjustments in their search functions to generate songs or playlists with varied inputs such as BPM, energy, acousticness, danceability, and many other interesting variables. I was ambitious and planned out a whole playlist generator complete with an album cover based off of a Pokemon dictated by the time of day and a playlist title and description based off of a Poem generated by the weather condition.
It was very ambitious and I spent hours trying to authenticate Spotify on my website (which I ended up doing) but did not get past that point of passing the authentication and search values through the API. I shifted gears and came up with a new game plan: a ridiculous hodgepodge of nonsensical inputs that output something that looked like a post on a feed. It includes image, a smaller profile image, profile name, and profile description, generated from a domino of other APIs.

The Chain

After writing out the code (mostly nesting APIs), I was able to get a few outputs:

In this example, the data I’m gathering is as follows:

Weather description: overcast clouds
The value used as a searchterm for MetAPI. I chose a random art image from that list

Wind Degree: 244
This value is divided by 12 and returns an integer anywhere between 1 to 30. The int is used in the PokemonAPI from 30 possible pokemon characteristics. Would return pokemon personality “Likes to fight”

Density: 10
This value is divided by 10 and returns an integer anywhere between 1 to 10. This int is used to pick from 10 possible pokemon colors. From the colors, a random pokemon is chosen from this pokemon list. Would return pokemon name “ribombee”

Year: 1993
This is the year the chosen art image was assessed. I subtract the years since the art piece has been assessed (e.g. 30) and use this value in poetryDB API to find poems with 30 lines. I choose one at random from the list, in this case first line of poem: “Streets of the roaring town,”

Putting it Together

Following (generally) the Figma sketch, I put together the different outputs and create this webpage that gives different responses on refresh depending on different weather conditions.
